# Progressive Web App (PWA) Implementation

This project now supports Progressive Web App functionality, allowing users to install the website as a native-like application on their devices.

## Features

- ✅ Offline support with service worker caching
- ✅ Installable on mobile and desktop devices
- ✅ Fast loading with cached assets
- ✅ Native app-like experience
- ✅ Responsive design for all screen sizes

## What's Been Added

### 1. Configuration Files

- **next.config.ts**: Configured with `next-pwa` plugin
- **public/manifest.json**: Web app manifest with metadata and icons

### 2. Generated Assets

- **public/icons/**: App icons in multiple sizes (72x72 to 512x512)
- **public/screenshots/**: Directory for app screenshots (optional)

### 3. Service Worker

Automatically generated by `next-pwa` during build:

- `public/sw.js`: Service worker file
- `public/workbox-*.js`: Workbox library files

## Testing the PWA

### Development Mode

PWA is disabled in development mode by default. To test:

```bash
npm run build
npm start
```

Then visit `http://localhost:3000` and check for the install prompt.

### Production Build

1. Build the project:

   ```bash
   npm run build
   ```

2. Start the production server:

   ```bash
   npm start
   ```

3. Open Chrome DevTools → Application tab → Manifest
   - Verify manifest is loaded correctly
   - Check icons are displayed
   - Test "Install" button

### Testing Checklist

- [ ] Manifest loads without errors
- [ ] All icons display correctly
- [ ] App can be installed on device
- [ ] App launches in standalone mode
- [ ] Offline functionality works
- [ ] Theme color matches brand (#1e40af)

## Customizing Icons

The current icons are placeholders (copies of logo.png). For production:

### Option 1: Online Generator (Recommended)

1. Visit https://realfavicongenerator.net/pwa/
2. Upload `public/logo.png`
3. Set background color: `#1e40af` (brand blue)
4. Download and replace icons in `public/icons/`

### Option 2: Use Script

```bash
npm run generate-icons
```

Note: This creates placeholder icons. Manual resizing is recommended for best quality.

### Option 3: Design Tool

Create custom icons using Photoshop, Figma, or similar tools with these sizes:

- 72x72, 96x96, 128x128, 144x144, 152x152, 192x192, 384x384, 512x512

## Manifest Configuration

Key settings in `public/manifest.json`:

```json
{
  "name": "SVI - Shiv Vardhan Infrastructure",
  "short_name": "SVI",
  "theme_color": "#1e40af",
  "background_color": "#ffffff",
  "display": "standalone"
}
```

To customize:

- **name**: Full app name shown on splash screen
- **short_name**: Name shown under app icon
- **theme_color**: Browser toolbar color
- **background_color**: Splash screen background
- **display**: How app appears (standalone, fullscreen, etc.)

## Service Worker Caching

The PWA automatically caches:

- Static assets (JS, CSS, images)
- Pages for offline viewing
- API responses (configurable)

Cache strategy: Network First with fallback to cache.

## Troubleshooting

### App Not Installing

- Ensure you're on HTTPS (or localhost for testing)
- Check manifest.json is valid (Chrome DevTools → Application)
- Verify service worker is registered
- Clear browser cache and reload

### Icons Not Showing

- Check icons exist in `public/icons/`
- Verify paths in manifest.json are correct
- Ensure icons are PNG format
- Regenerate icons if needed

### Offline Not Working

- Service worker only activates after first load
- Check Cache Storage in DevTools
- Verify pages are being cached
- Rebuild and restart server

## Deployment Notes
